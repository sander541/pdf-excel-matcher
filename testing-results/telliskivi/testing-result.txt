Key points
- Code format mismatch: Excel has detailed codes like VMPU‑3.1 … VMPU‑3.5, but
  the PDF shows many “VMPU‑3”. Treating these as the same would need special
  rules and is risky for a simple tool.
- Labels on the PDF are drawings, not selectable text.
- Many labels have lines crossing through them, which breaks OCR reading.
- Labels are small, so OCR needs strong zoom. Together, this makes OCR results
  unreliable on this file and can make processing quite slow.
- Annotations get messy because the same base code appears on different floors (probably could improve that part).
